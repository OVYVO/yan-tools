# UI 修改需求文档（v1.1）

**项目**：个人前端工具站  
**关联**：`prd.md`、`ui-style-spec.md`  
**文档性质**：在现有实现基础上的界面与交互调整说明，供设计与开发对齐验收。

---

## 1. 详情页（工具页）

### 1.1 各区域内容宽度一致

**现状问题**：工具操作条与正文容器最大宽度不一致。例如 `ToolPage.vue` 中 `#tool-actions` 使用 `max-w-5xl`，而 `ToolPageTemplate.vue` 主区域为 `max-w-[min(100%,1440px)]`，易造成视觉「错层」。

**需求**：

- 详情页内 **所有横向区块**（含：面包屑与工具介绍区、错误提示、工具主体 `slot`、相关工具区）使用 **同一套水平约束**。
- 建议与全站内容区对齐：采用与 `bento-container` / 首页一致的 **`max-w-[min(100%,1440px)]`**（或抽成单一布局类名），左右 **`px` 与首页 `bento-container` 一致**（当前为 `px-4 sm:px-6`）。
- 工具页内若存在 **独立窄栏**（如仅表单居中），须在文档或代码注释中说明例外；默认 **不允许** 无理由混用 `max-w-5xl`、`max-w-3xl` 等与主内容不一致的宽度。

**验收**：

- 从页顶到页底，主内容左边缘、右边缘在统一断点下对齐（允许工具内部代码区单独横向滚动，但整体卡片/栅格外轮廓仍与主宽度一致）。

### 1.2 同行多卡片时，间距与首页一致

**基准**：首页工具列表使用 Uno 快捷类 **`bento-tool-grid`**（`gap-5`，`md:gap-5`，与 `uno.config.ts` 注释一致）。

**需求**：

- 详情页 **「相关工具」**（及未来若在详情页增加的同类多卡片区）必须使用与首页 **相同的栅格与 gap 令牌**，即复用 **`bento-tool-grid`**（或与其 **完全等价** 的 `grid` + `gap` 组合，禁止另起一套更紧/更疏的间距）。
- 若一行展示多张 `ToolCard`，列跨度规则与首页 **「最新上线」等区块** 保持一致（当前 `ToolCard` 为 `md:col-span-6 lg:col-span-3`）。

**验收**：

- 首页与详情页相关工具区并排对比时，**卡片间距视觉一致**；无「详情页更挤或更散」的主观差异。

### 1.3 收藏入口统一放入「工具介绍」卡片

**现状**：收藏按钮位于 `ToolPage.vue` 的 `#tool-actions` 条（工具正文之上），与 `ToolPageTemplate` 顶部的介绍区分属两块区域。

**需求**：

- **详情页所有与「收藏」相关的图标与操作**（含：收藏 / 已收藏状态切换、无障碍文案）一律放在 **`ToolPageTemplate` 顶部介绍区**（即当前带面包屑、标题、摘要、标签的 **`bento-surface` 卡片**）内。
- 推荐布局：在标题行 **右侧** 或 **摘要区右上角** 放置 **图标按钮**（或「图标 + 文案」），与标题对齐，小屏时允许换行但仍在 **同一卡片** 内。
- 移除（或不再使用）`ToolPage.vue` 中独立于介绍卡片之外的收藏条；`#tool-actions` 若仍用于锚点跳转，可仅保留「快捷操作」等非收藏内容，或整体改为跳转到介绍区内对应位置（按实现二选一，以 **无重复收藏入口** 为准）。

**验收**：

- 全页仅 **一处** 收藏交互；位置在工具介绍卡片内；键盘与读屏可聚焦，状态与 `useFavorites` 逻辑不变。

## 2. 全局（顶栏与页脚）

### 2.1 页脚不使用卡片样式

**现状**：`AppFooter.vue` 外层使用 `bento-surface`（圆角大块、阴影、玻璃感）。

**需求**：

- 页脚改为 **扁平条带式**：**无** 大块圆角卡片容器、**无** 与 `bento-surface` 等价的强阴影「浮起」感。
- 允许：顶部分割线（`border-t`）、与页面背景略区分的底色或纯透明、内边距与排版保持清晰。
- 与 `ui-style-spec` 中「页脚信息密度低、不与 Bento 抢视觉」一致。

**验收**：

- 页脚视觉上为 **页面基底延伸**，而非中间悬浮的一张卡。

### 2.2 顶部导航不使用卡片样式

**现状**：`AppHeader.vue` 内层使用 `bento-surface` + `rounded-ui-lg`，呈现整块导航卡片。

**需求**：

- 顶栏改为 **非卡片化**：去掉包裹整条导航的 `bento-surface` 式圆角底托；可采用 **底边线**、**半透明条** 或与背景融合的 **sticky 顶栏**，但 **不出现** 与主内容区同级的「大圆角阴影卡片」观感。
- 保留：`sticky`、品牌区、导航链接、主题切换等 **功能与信息架构不变**。

**验收**：

- 首屏与滚动后，顶栏均为 **扁平导航条** 语义，不与下方 Bento 内容块混淆。

### 2.3 替换新的图标

**现状**：站点多处使用 **Carbon** 图标集（如 `i-carbon-tools`、`i-carbon-search`、`i-carbon-star` 等，见 `AppHeader.vue`、`HomePage.vue` 等）。

**需求**：

- 全站导航、页脚、首页关键入口、收藏星标等 **统一迁移到同一套新图标方案**（仍通过 UnoCSS Icon 或 SVG 组件，二选一由实现确定）。
- **必须替换的典型位置**：站点 Logo 区、搜索、主题下拉装饰、收藏相关、页脚外链旁若存在图标、首页「查找工具」与分类箭头等。
- **规范**：同一语义在全站 **只使用一种图标图形**；尺寸与笔画粗细在顶栏与内容区之间 **分级一致**（可定义 `icon-sm` / `icon-md`）。

**验收**：

- 无混用两套图标族；浅色/深色下图标可读；焦点环不被裁切。

### 2.4 全局背景：粒子连线动效与鼠标交互

**现状**：`main.css` 中 `.bento-backdrop` 已为固定装饰层（光晕 + 点阵网格），无动态粒子与指针交互。

**需求**：

- 在 **全站页面背景**（内容区之下、与现有 `.bento-backdrop` 同层或合并为一层）增加 **粒子 + 连线** 动效：
  - 若干 **粒子点** 在视口内缓慢漂移或微幅运动；
  - 当两粒子距离小于阈值时，绘制 **连线**（线段透明度随距离衰减，远处不断线或弱化）；
  - 整体风格 **低对比、轻量**，不抢占前景卡片与文字注意力。
- **鼠标位置交互**（至少实现其一，推荐组合）：
  - **影响范围**：以指针为圆心，一定半径内粒子受到 **排斥、吸引或速度扰动**（具体曲线可设计，需体感自然、不抖动）；
  - **连线增强**：指针附近的粒子对之间连线可 **略增亮或略加粗**，离开指针区域后平滑恢复；
  - **离开视口 / 无移动**：可缓慢衰减到默认漂移状态，避免指针消失后画面突变。
- **层级与命中**：
  - 动效层 **`pointer-events: none`**，不拦截点击、拖拽、选中文本；
  - `z-index` 低于 `.bento-shell`（或主内容包装），确保 **所有可交互控件** 在动效之上。
- **主题**：浅色 / 深色模式下粒子、连线颜色 **分别适配**（或统一用低透明度中性色），与 `--ui-bg-base` 协调，禁止出现刺眼的纯白高亮线铺满屏。
- **性能**：
  - 使用 `requestAnimationFrame` 单路更新；粒子数量与连线判定需 **可配置**，默认在常见笔记本分辨率下保持流畅（目标 **≥ 50fps** 或体感无卡顿）；
  - 窗口 `resize` 时重建或修正粒子分布，避免拉伸错位；
  - 可选：`document.visibilityState === 'hidden'` 时 **暂停** 动画以省电。
- **无障碍与系统偏好**：
  - 当用户开启 **`prefers-reduced-motion: reduce`** 时：**关闭** 粒子运动与连线闪烁，改为 **静态背景**（保留极淡点阵/纯色即可）或完全关闭该层，与 `ui-style-spec` 动效原则一致。
- **实现方式**：`Canvas 2D` / `WebGL` / 纯 CSS（若可达成）均可，需在代码中 **集中封装**（如 `ParticleBackdrop.vue` + composable），避免逻辑散落在 `App.vue`。

**验收**：

- 全站各路由页背景一致出现该效果（或按 reduced motion 规则关闭）；
- 移动鼠标时背景有 **可感知的跟随/扰动**，且 **不影响** 页面操作与滚动；
- 深浅色切换后动效仍 **柔和可读**，无闪烁硬切；
- 开启「减少动态效果」后 **无持续粒子动画**。

## 3. 首页（工具卡片）

### 3.1 不同颜色的渐变背景

**需求**：

- 每个 **工具卡片**（`ToolCard`）在保持现有信息层级的前提下，增加 **可区分的渐变背景**（不同卡片色相不同，避免全站单一色）。
- **实现策略**（任选其一，需全站统一）：
  - **按索引轮换**：`nth-child` 或根据 `slug` 哈希映射到预设的 4～8 套渐变，保证相邻卡片易区分。
  - **按分类/元数据**（若后续扩展）：从 `category` 映射色板。
- **对比度**：渐变上仍需保证 **标题、摘要、标签** 可读，符合 WCAG AA；深色模式下使用 **单独一套** 渐变或降低透明度，避免灰字糊在背景上。
- 与现有 `tool-card` 左侧竖条高亮（`before` 渐变）协调：可保留、弱化或合并进新背景方案，以 **不杂乱** 为准。

**验收**：

- 同一列表内卡片背景 **明显可区分**；任意主题下文字清晰。

### 3.2 交互效果

**需求**：

- 在现有 `hover` 边框与阴影基础上，增加 **可感知的微交互**，例如：
  - 轻微 **上移**（`translate-y`）或 **缩放**（`scale`）；
  - 渐变 **亮度/色相** 的过渡；
  - 内部装饰（如光晕）随悬停显现。
- **约束**：时长建议 **150～200ms**，缓动与全站其他组件一致；遵守 **`prefers-reduced-motion: reduce`** 时减弱或关闭位移动画（与 `main.css` 中现有策略一致）。

**验收**：

- 悬停与键盘 `:focus-visible` 均有清晰反馈；动效不眩晕、不遮挡文字。

## 4. 实现与文档关系

| 主题 | 建议改动文件（参考） |
|------|----------------------|
| 详情宽度与收藏位置 | `ToolPage.vue`、`ToolPageTemplate.vue` |

| 图标替换 | 上述组件 + `uno.config.ts` / Icon 预设 |
| 全局粒子连线背景 | `App.vue` 或布局壳、`main.css`（与 `.bento-backdrop` 协调）、新建 `ParticleBackdrop` 类组件 + 可选 `usePointer`/`useRaf` |
| 首页卡片渐变与动效 | `ToolCard.vue`、`uno.config.ts` 或 `main.css` 令牌 |

**说明**：本文件为 **需求** 文档；具体色值与图标包名可在设计走查后填入附录，不改变上述验收条款。
