# UI 样式需求文档（Bento Grid）

**项目**：个人前端工具站  
**技术栈**：Vue 3 + Vite + TypeScript + UnoCSS  
**关联文档**：`prd.md`  
**文档目的**：约束全站视觉与布局，保证风格统一，并以 **Bento Grid（便当盒式网格）** 作为首页与内容聚合页的核心布局语言。

---

## 1. 设计目标

| 目标 | 说明 |
|------|------|
| **风格统一** | 全站共用同一套设计令牌（颜色、圆角、间距、字体层级、阴影），禁止页面各自为政的「一次性样式」。 |
| **Bento 识别度** | 主要浏览场景（首页、分类聚合、搜索摘要区）采用不规则但可预测的网格块，形成清晰的信息分区。 |
| **工具站气质** | 偏现代、克制、偏「生产力」；避免过度装饰与强营销风。 |
| **可扩展** | 从 30+ 到 100+ 工具时，卡片与网格规则仍适用，无需改版心智力。 |

---

## 2. 设计原则

1. **块即信息单元**：每个 Bento 单元格承载一类信息或一个行动入口，避免单格堆砌过多层级。
2. **主次分明**：全站最多一个「视觉焦点块」（如搜索或推荐），其余为次级块；焦点块可通过跨列/跨行体现。
3. **呼吸感**：单元格内边距与块间距统一来自间距令牌，避免「贴边」与「随机 margin」。
4. **浅色 / 深色等价**：同一令牌在两种主题下对比度与层级一致，禁止仅浅色下可用的灰阶。
5. **动效克制**：悬停与聚焦用于反馈，时长与曲线全站一致；禁止大面积持续动画干扰阅读。

---

## 3. Bento Grid 规范

### 3.1 定义

- **Bento Grid**：基于 CSS Grid 的响应式布局，由多个 **矩形卡片（Cell）** 组成；卡片可 **跨列、跨行**，形成「便当盒」分格效果。
- **Cell（单元格）**：最小展示容器，对应一个 `ToolCard`、分类入口、搜索栏容器、或「最近使用」列表外壳等。

### 3.2 网格参数（桌面优先）

| 项 | 建议值 | 说明 |
|----|--------|------|
| 列数 | **12** | 便于 1/2、1/3、1/4、1/6 等分栏；焦点块常用 `span 6–12`。 |
| 列间距（gap） | **16px（`gap-4`）** | 全站 Bento 区域统一；特殊全宽分隔带可用 `gap-6` 需在设计评审中注明。 |
| 行间距 | 与列间距 **相同** | 保持方格节奏一致。 |
| 最小行高 | 由内容决定；**空状态** 时最小高度 **≥ 120px** | 避免扁条块破坏 Bento 观感。 |

### 3.3 单元格跨度（Span）约定

- **小型工具卡片**：默认 **列 span 3**（12 列下每行 4 个）或 **span 4**（每行 3 个）；全站选定一种作为默认，**分类页与首页一致**。
- **中型块**（如「分类入口」「单条推荐」）：**span 4–6**。
- **大型块**（搜索、Hero、精选专题）：**span 8–12**，可 **row span 2** 以形成视觉锚点。
- **侧边信息**（如「最近使用」摘要）：桌面端 **span 4**，主内容 **span 8**；移动端 **单列堆叠**，顺序按信息优先级排列。

### 3.4 响应式断点

| 断点 | 宽度（约） | Bento 行为 |
|------|------------|------------|
| `sm` | ≥ 640px | 可 2 列稀疏 Bento。 |
| `md` | ≥ 768px | 至少 2 列，推荐开始 12 列网格。 |
| `lg` | ≥ 1024px | 完整 12 列 Bento，允许跨列布局。 |
| `xl` | ≥ 1280px | 最大内容宽度 **限制在 1280px–1440px** 内居中，避免卡片在超宽屏过度拉伸。 |

**移动端（< sm）**：单列栈式布局；**禁止** 保留复杂跨行，避免阅读顺序混乱。可将多段 Bento 拆为多个纵向 Section，每 Section 内仍保持统一 `gap`。

### 3.5 视觉统一（Bento 专用）

- 所有 Cell 外层使用 **同一卡片容器规范**（圆角、边框/阴影、背景），见第 5 节。
- **禁止** 相邻两格使用冲突的边框粗细或圆角规格。
- 跨多列的大块与小块 **顶部对齐基线** 可选：大块标题与小块标题使用 **同一标题层级令牌**。

## 4. 全局设计令牌（Design Tokens）

### 4.1 圆角

| 令牌 | 值（建议） | 用途 |
|------|------------|------|
| `radius-sm` | 6px | 按钮、小标签、输入框 |
| `radius-md` | 10px | 默认卡片、下拉、模态 |
| `radius-lg` | 14px | Bento 主卡片、首页焦点块 |
| `radius-full` | 9999px | 胶囊按钮、头像占位 |

全站 **Bento 卡片统一使用 `radius-lg` 或 `radius-md` 二选一**，不可混用。

### 4.2 间距

- 基准单位：**4px**；页面边距：**16–24px**（移动端偏小、桌面偏大）。
- 卡片 **内边距**：**16px** 或 **20px**（全站一致）。
- Section 间距（首页大区块之间）：**32–48px**。

### 4.3 字体与字阶

| 层级 | 用途 | 建议 |
|------|------|------|
| `text-page-title` | 页面主标题 | 单页唯一，加粗 |
| `text-section` | Bento 区块标题 | 小于主标题，字重 semibold |
| `text-body` | 正文、描述 | 常规行高 1.5–1.6 |
| `text-caption` | 辅助、元信息 | 略小 + 次要色 |

**数字与等宽内容**（代码、哈希、时间戳）使用 **等宽字体栈**，与正文区分。

### 4.4 阴影与层次

- **默认卡片**：浅阴影或仅边框（二选一为主风格，全站统一）。
- **悬停**：阴影略升或边框对比略升，**同一交互深度**。
- **焦点环（focus-visible）**：键盘导航可见，颜色与主色关联，**禁止** 仅依赖 `outline: none` 而无替代。

---

## 5. 色彩与主题

### 5.1 语义色

| 语义 | 用途 |
|------|------|
| `bg-base` | 页面背景 |
| `bg-elevated` | 卡片、浮层背景 |
| `border-subtle` | 分割线、卡片边 |
| `text-primary` | 主文案 |
| `text-secondary` | 次要说明 |
| `text-muted` | 占位、禁用感文案 |
| `accent` | 主操作、链接、选中态 |
| `danger` | 错误、破坏性操作 |

### 5.2 主题模式

- 支持 **浅色 / 深色 / 跟随系统**（与 `prd.md` T-017 一致）。
- 主题切换后，Bento 卡片与导航的 **对比度** 需满足 WCAG AA（正文与背景）。

---

## 6. 组件与页面映射

### 6.1 全局壳层

- **顶栏**：固定高度或紧凑模式二选一；与内容区间距使用 Section 间距令牌。
- **页脚**：信息密度低，不与 Bento 抢视觉权重。

### 6.2 首页（Bento 主战场）

建议区块顺序（可微调，但 **每块应为独立 Bento Section**）：

1. **搜索 / Hero**（大格，跨多列）
2. **推荐或精选**（中到大格）
3. **分类入口**（多格均分）
4. **最近使用 / 收藏**（中格或条带格）
5. **最新工具**（小卡网格，与分类页卡片规格一致）

### 6.3 分类页与搜索页

- 列表仍以 **统一 `ToolCard`** 呈现；可采用 **较密的网格**（仍基于同一 12 列或自动填充 `grid`），但 **单卡样式与首页一致**。
- 筛选、排序置于 **单独一行或独立 Cell**，不挤占卡片内部。

### 6.4 工具详情页（`ToolPageTemplate`）

- **可不使用 Bento**，以保持操作区稳定；若使用，仅用于「说明 + 相关推荐」等次要信息区，**输入/输出主栏保持单列或主辅两列**，避免干扰任务流。

### 6.5 公共组件（与 `prd.md` T-011 对齐）

- `ToolCard`、`分类卡片`、`EmptyState`、`UiTextarea`、主按钮/次按钮均在令牌内实现，**禁止** 在单工具内覆盖全局字号与圆角。

## 7. 交互与动效

| 场景 | 要求 |
|------|------|
| 卡片悬停 | 时长 **150–200ms**，缓动统一（如 `ease-out`） |
| 主题切换 | 可淡入淡出 **≤ 300ms**，避免闪烁硬切 |
| 页面进入 | 可选轻微 stagger，**仅首页/列表首屏**，且可关闭或减少动效（尊重系统「减少动态效果」） |

---

## 8. 可访问性

- 语义化标题层级：`h1` 每页一个；Bento 区块标题用 `h2`/`h3` 不跳级。
- 图标按钮需 **`aria-label`** 或可读文本。
- 键盘：Bento 内可聚焦元素顺序与视觉顺序一致。
- 对比度：浅色/深色均达标；链接不仅依赖颜色，可加下划线或图标。

---

## 9. 实现约束（UnoCSS）

1. 颜色、圆角、间距优先使用 **Shortcuts 或主题变量** 集中定义，避免魔法数字散落。
2. Bento 布局使用 **CSS Grid**（`grid grid-cols-12` 等），与子项 `col-span-*` / `row-span-*` 配合；复杂顺序用 `order-*` 仅在移动端调整。
3. 新增页面时先 **对齐本文件与 `prd.md`**，再写局部样式。

---

## 10. 验收清单（摘要）

- [ ] 首页、分类页、搜索列表的 **卡片视觉一致**，圆角与阴影符合令牌。
- [ ] 桌面端存在 **可识别的 Bento 布局**（跨列/跨行至少一处合理运用）。
- [ ] 移动端 **单列** 清晰，无横向意外滚动（除表格/代码横向滚动容器内）。
- [ ] 浅色/深色 **全站关键组件** 无「看不见的文字」或「融为一体的边框」。
- [ ] 焦点态与悬停态在全站 **一致可预期**。
